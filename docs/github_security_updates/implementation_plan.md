# セキュリティ脆弱性への対応：依存関係のアップデート計画

GitHubのセキュリティアラートで報告された脆弱性に対応するため、以下の依存関係をアップデートします。

## 概要
複数のNode.jsおよびRustのライブラリに脆弱性が検出されました。これらを推奨される安全なバージョンにアップデートし、アプリケーションの安全性を確保します。

## 変更内容

### Node.js 依存関係 (package.json / package-lock.json)

以下のパッケージをアップデートします。直接の依存関係にあるものは `package.json` を更新し、間接的なものは必要に応じて `npm install` や `npm update` で対応します。

- **`cookie`**: `^0.6.0` -> `~> 0.7.0`
- **`@sveltejs/kit`**: `^2.21.1` -> `^2.49.5`
- **`devalue`**: `^5.3.2` -> `^5.6.2`
- **`svelte`**: `^5.0.0` -> `^5.51.5`
- **`rollup`**: `^4.34.9` -> `^4.59.0`

### Rust 依存関係 (src-tauri/Cargo.lock)

Rustの依存関係は間接的なものが多いため、`cargo update` を使用して、以下の推奨バージョンを満たすように更新します。

- **`glib`**: `~> 0.20.0` (現在 0.18.5)
- **`bytes`**: `~> 1.11.1` (現在 1.10.1)
- **`time`**: `~> 0.3.47` (現在 0.3.37)

---

## 検証計画

### 自動チェック
1. **Node.js タイプチェック**:
   - コマンド: `npm run check`
   - 目的: SvelteKitやSvelteのアップデートによる型定義の不整合がないか確認します。
2. **Node.js ビルド**:
   - コマンド: `npm run build`
   - 目的: Vite/Rollupのアップデートによるビルドエラーがないか確認します。
3. **Rust ビルド**:
   - コマンド: `cd src-tauri && cargo build`
   - 目的: Rust依存関係のアップデートによるコンパイルエラーがないか確認します。

### 手動確認
- アプリケーションを起動し、主要な機能（ファイル読み込み、計算処理など）が正常に動作することを確認します。
- コマンド: `npm run tauri dev`
